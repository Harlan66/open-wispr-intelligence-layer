# open-wispr-intelligence-layer

[English README](./README.md)

一个面向本地语音转录系统的 **智能增强层（intelligence layer）**。

它的核心目标不是单纯做“混合审查”，而是解决这样一个更本质的问题：

> **纯本地文字转录通常不够智能。**

比如：

- 技术术语识别不稳定
- 中英混合表达容易错
- 很难区分“真正纠错”与“语义改写”
- 如果直接从用户编辑里学习，很容易污染词表
- 一旦语义复核层依赖远程模型，又容易被网络/供应商抖动拖垮

这个项目给出的不是单点技巧，而是一套可落地的方法论：

## 核心思路

不是走两种极端：

### 极端 1：纯本地
优点：
- 稳
- 快
- 隐私好

缺点：
- 不够聪明
- 很难处理边界语义
- 很难安全进化词表

### 极端 2：纯云端
优点：
- 模型判断力强

缺点：
- 容易贵
- 容易脆
- 容易夜里任务全挂
- 容易把“静默后台能力”做成“需要人盯着的工作流”

所以更合理的方案是：

> **local-first + selective intelligence**

也就是：
- 主转录链仍然本地
- 大量样本先由本地逻辑稳定粗筛
- 只把少量高价值、强歧义候选交给 agent 做语义复核
- 出问题时具备自动恢复能力

---

## 这套架构在解决什么

它主要解决的是：

### 1. 本地转录“不够智能”
本地 ASR 很擅长把声音转成文字，
但不擅长判断：
- 这是术语归一化，还是句子重写？
- 这是局部修正，还是上下文污染？
- 这是应该学习的样本，还是该排除的噪音？

### 2. 自动学习“容易学坏”
如果把用户编辑后的文本直接当作纠错来源：
- URL 粘贴
- 上下文拼接
- 风格改写
- 语义重写
都会把词典带歪。

### 3. 远程增强层“不够稳”
如果所有夜间任务都依赖远程模型，一旦：
- API 供应商抖动
- 网络瞬断
- 定时任务时段撞上高峰
整条链就会变脆。

## 方案结构

### 第一层：本地转录（primary path）
继续由 `open-wispr` 负责主路径：
- 本地模型
- 本地热键
- 本地实时输入

这保证了：
- 日常使用不依赖云端
- 速度与隐私都可控

### 第二层：本地夜间粗筛（stability layer）
本地脚本负责：
- 读取最近 24 小时 transcript/edit 样本
- 做保守分类：
  - `accepted`
  - `local_correction`
  - `rewrite`
  - `style_edit`
  - `unclear`
- 过滤明显脏样本：
  - URL 注入
  - 上下文拼接
  - 大段语义改写
  - 模糊边界项
- 生成本地产物：
  - `nightly-review-latest.md`
  - `review_queue.jsonl`
  - `remote-review-input.jsonl`

这一层的关键词是：
- 稳定
- 安静
- 低成本
- 防污染

### 第三层：agent 语义复核（intelligence layer）
真正需要智能的，不是全量样本，而是少量难判断样本。

所以 agent 只看小批量候选，用来判断：
- 这是不是安全的局部纠错？
- 这是术语归一化还是重写？
- 是否可以自动写回？

这一层不是为了替代本地层，而是为了：

> **把模型智力只用在值得用的地方。**

### 第四层：网络恢复自动补跑（resilience layer）
如果远程复核因为：
- 网络问题
- 上游 API 抖动
- 调度失败
而没跑成，
则由本地 watcher 在网络恢复后自动补跑。

也就是说：

> 把“远程失败”视作可恢复的运行状态，而不是特殊灾难。

## 优势

### 稳
主链路仍然本地。

### 聪明
语义判断交给 agent，而不是强迫本地规则脚本装聪明。

### 安静
夜间运行，不打扰聊天。

### 防污染
默认严格保守，宁可漏，不轻易学坏。

### 可恢复
网络恢复后自动补跑，不把临时故障当永久失败。

---

## 劣势

### 比纯本地更复杂
多了一层复核和恢复机制。

### 比纯云端更保守
不是所有“看起来可学”的内容都会立即进入词表。

### 需要可观测性
如果没有日志/产物文件，很容易变成“以为在跑，其实没跑”。

---

## 适合谁

如果你想做的是：
- 本地语音输入
- 中英混合术语场景
- 技术写作/编码场景
- 静默后台进化
- 不希望人工频繁干预

这套方法比“纯本地”或“纯云端”都更平衡。

---

## 一句话总结

这个项目最准确的定位是：

> **为本地语音转录系统补上一层“智能但稳定”的增强层。**

`open-wispr` 是当前参考对象，但方法本身可以泛化到其他本地转录/听写工具。
